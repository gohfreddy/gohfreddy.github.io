Reach+ Game Redesign

Feb â€“ Mar 2014


Holland Bloorview Kids Rehabilitation Hospital developed a game which interfaces with Microsoft's Xbox Kinect to promote arm extension rehabilitation tasks primarily for children with cerebral palsy (CP). They partnered with University of Waterloo to evaluate, redesign, and provide design recommendations based on the initial game screens and questionnaires.

Overview

Problem: The challenge was to evaluate and redesign the game experience and the usability testing protocol

Client: Holland Bloorview Kids Rehab - children's rehabilitation hospital based in Toronto

Role: Our team would work on each part individually, provide critiques, and combine ideas for a final version. In particular, I worked on the low fidelity prototype.

Note: This was a term project for the User Centered Design course (SYDE 348). My group members included Alan An, Michael Bustamante, Angelica Escareno, and Catherine Maritan.

User Research

User Groups

  1. Therapists - set up the game sessions and monitor progress
  2. Children - engage with the game/rehabilitation

Limitation

There was a lack of accessibility to real user groups and were instructed to complete evaluations by having peers role-play as users

Original Design

Start Screen

  • does 'Start' button enter gameplay directly?

Profile Selection

  • icons are small and unclear

Settings

  • unconventional form elements
  • missing descriptions and non-discoverable

Level Selection

  • selecting level unexpectedly begins game right away

In Game Screen

  • unclear what the bar in the middle refers to

Personas

Usability Issues

Therapist

  • lack of flow and linearity in setup
  • elements and descripitions used were unconventional and unclear
  • commonly lead to trial and error

Child

  • instructions were heavy with text and unlikely to be read
  • "Why can't I grab the food?"
  • feedback and response was not obvious to reinforce action or promote engaging gameplay

Project Goals

  1. Linearize the workflow and reduce redundant clicks
  2. Create a familiar settings menu with more discoverability
  3. Give child more control
  4. Provide intuitive and responsive feedback
Features

Linearize Workflow

  • can't start game directly form start screen
  • must have profile loaded to play (and track stats)
  • takes two clicks and a password entry to get into settings

Familiar Settings Menu

  • added concept of presets to reduce setup time
  • large and obvious buttons with a preview of settings when selected
  • edit presets by pressing familiar settings icon
  • intuitive settings adjustment
  • hover over setting to prompt hint/description box/modal

Child Control

  • allow child and Kinect to orient themselves to each other
  • when child is ready, he/she places hands over ready button to start game

Responsive Feedback

  • original design had poor feedback to when an appropriate elbow extension angle was achieved
  • coloured overlay about elbow joint indicates that child met the angle threshold or not
  • when elbow angle is greater or equal to threshold, the overlay turns green and the target object can be grabbed
  • another responsive feedback could include pet emotions when being fed
Usability Testing

Limitation

Walkthroughs and Heuristic Evaluation methods are typically done by experts in the field

Due to lack of accessibility, these evaluations were done by two student peers from other teams

Walkthrough

Evaluators asked to go through wireframes using think-aloud

Comments and observations were recorded while noting any critical incidents

Heuristic Evaluation

Heuristic/usability goals were listed in a checklist and user emotions and comments were recorded

Feedback

  • angle threshold visualization is great
  • presets seem like an effective way to minimize setup errors
  • some back button controls require rework
  • quit button is too close to other buttons
  • more game rule description required

Final Evaluation

Constraints

Usability testing was done in-class and limited by time (20 minutes) and space (section of classroom)

Limitation

Does not discover true usability problems as no real users were involved with the evaluation

Evaluation Goals

  1. Complete walkthrough of prototype with specific tasks
  2. Understand major points of confusion
  3. Gather user reaction and emotion towards product

Reflection

This project was a great exercise for prctising a logical design process for redesigning a game. Usability testing was important in driving design decisions.

We hope that our design recommendations and findings would be taken into consideration in future development of this game.

We understand that some issues lie beyond UI and usability. The game concept is probably too simple to keep users engaged for a long period of time or subsequent sessions. Additional game mechanics should be explored for feasibility such as: achievement levels aligning with rehabilitation goals, use of other movements, unlocking new pets, and use of other elements other than food.
