Reach+ Game Redesign

Feb – Mar 2014


Holland Bloorview Kids Rehabilitation Hospital developed a game which interfaces with Microsoft's Xbox Kinect to promote arm extension rehabilitation tasks primarily for children with cerebral palsy (CP). They partnered with the University of Waterloo to evaluate, redesign, and provide design recommendations based on the initial game screens.

Overview

Problem: The challenge was to evaluate and redesign the experience of game setup, game interaction, and the usability testing protocol

Client: Holland Bloorview Kids Rehab - children's rehabilitation hospital based in Toronto

Role: Our team would work on each part individually, provide critiques, and combine ideas for a final version. In particular, I worked on the design features and low-fidelity prototype.

Note: This was a term project for the User Centered Design course (SYDE 348). Group members: Alan An, Michael Bustamante, Angelica Escareno, and Catherine Maritan.

User Research

User Groups

  1. Therapists – set up the game sessions and monitor progress
  2. Children – engage with the game/rehabilitation

Original Design

In this game, the player is tasked with feeding their pet (who is positioned somewhere on the screen). Food items will appear on the screen, and the player must 'reach' to pick them up and deliver them to their pet.

Personas

Usability Issues

Therapist

  • lack of flow and linearity in setup
  • elements and descripitions used were unconventional and unclear
  • commonly led to trial and error

Child

  • instructions were heavy with text and unlikely to be read
  • "Why can't I grab the food?"
  • feedback and response was not obvious to reinforce action or promote engaging gameplay
Features

Linearize Workflow

  • can't start game directly form start screen
  • must have profile loaded to play (and track stats)
  • takes two clicks and a password entry to get into settings

Familiar Settings Menu

  • added concept of presets to reduce setup time
  • large and obvious buttons with a preview of settings when selected
  • edit presets by pressing familiar settings icon
  • intuitive settings adjustment
  • hover over setting to prompt description modal

Giving Child Control

  • allow child and Kinect to orient themselves to each other
  • when child is ready, he/she places hands over ready button to start game

Responsive Feedback

  • original design had poor feedback to know when an appropriate elbow extension angle was achieved
  • coloured overlay about elbow joint
  • when elbow angle is greater or equal to threshold, the overlay turns green and the target object can be grabbed
  • another responsive feedback could include pet emotions when being fed

Reflection

This project had couple limitations regarding the lack of accessibility to 1. real user groups for usability testing and 2. expert evaluators for the walkthroughs and heuristic evaluation methods. We were instructed to complete evaluations by having peers role-play as users or expert evaluators. This project was completed for course credit and having the entire class interview or evaluate real user groups would provide overwhelming stress.

Usability testing and feedback with a real users is critical for identifying real usability issues and there isn't really a way around this. This project was a good practice for following a logical design process but if I could repeat this project outside the contraints of the classroom, I would start with the real users.

We hope that our design recommendations and findings would be taken into consideration in future development of this game.

We understand that some issues lie beyond UI and usability. The game concept is probably too simple to keep users engaged for a long period of time or subsequent sessions. Additional game mechanics should be explored for feasibility such as: achievement levels aligning with rehabilitation goals, use of other movements, and unlocking new pets.

This was my first exposure to wireframing and I learned how to use wireframes to effectively communicate and express an idea. I would have liked to explore the user experience in the game play and in the Kinect environment further.
